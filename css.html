Style Sheets

Pre-processing

CSS pre-processing allows us to write DRY code that will be faster, more efficient and easier to maintain. We have chosen SCSS as our CSS pre-processor. SCSS (not SASS) is more powerful and offers more features than plain CSS.

Class Naming Convention

Write lower case, using dashes.

Correct: 

.some-class {}

Incorrect: 

.someClass {}
.some_class {}
.someclass {}
.SomeClass {}

Class Pre-fixing

  • .js- when hooking to a Javascript function. Do not apply style to a class prefixed with .js-.
  • .is- for stateful classes (often toggled), for example .is-hidden.

Declarations

  • Place Scss/css declarations in alphabetical order. This is to easily find what you are looking for, and to eliminate the risk of duplication.
  • Use variables as much as possible.
  • Wrap inside of so called ”egyptian brackets”.
  • Indent by 4 spaces.
  • Arrange pseudo classes like below.

As shown in this example: 

.some-class {
    color: $color;
    font-family: $font;
    padding: 1rem;
    width: (100% / 3);
    &:hover {
        color: darken($color, 5%);
    }
}

Nesting

Do not nest, it's an expensive operation. Always try to find a direct selector. Read more in the BEM section.

Correct:

.some-list {}
.some-list__item {}

Incorrect:

.some-list > div {
    ul {
        li.item {
        }
    }
}
