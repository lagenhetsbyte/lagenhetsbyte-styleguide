Semantic HTML

Semantics are the implied meaning of a subject, like a word or sentence. We use semantic HTML. HTML elements need to have their distinguishable structural roles. Proper semantics equals a more searchable content which leads to a better search engine ranking. We also increase accessibility.

HTML5 offers, amongst others, these semantic elements: 

<article>
<aside>
<audio>
<canvas>
<details>
<figcaption>
<figure>
<footer>
<header>
<main>
<nav>
<section>
<summary>
<time>
<video>

Line Length

Avoid lines longer than 80 characters. The practice of keeping code to ~80 columns might be old, but there are still valid reasons to keep the rule:

  • To view multiple source windows side-by-side or using a side-by-side diff viewer.
  • To improve readability. Narrow code can be read quickly without having to scan your eyes from side to side.

Please note: When a statement or element will not fit on a single line, it may be necessary to break it. Place the break after an operator or attribute, ideally after a comma (if there is one). The following line/lines should be indented. 

<input type="email" name="email" id="formEmail" class="form__input"
    placeholder="name@example.com" required autocomplete="email">
