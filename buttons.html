Buttons

All actions are not equal. By using visual distinction, we are helping the users to make the right choices. The visual presentation of actions should match their importance.

Markup

Call the button element with <a class="button"></a>.
Then add the type of button as an additional class: button--blue.
The default size is a large button.
However, if there is a need to adjust the button size, add class button--medium or button--small.

Button or link?

Every action on a website falls under two different categories:

1)

Actions where users affect the website’s back-end or front-end.
For example:
  • Sign-up and purchase actions
  • Creating a new account
  • Ceating new blogpost
  • Submitting a comment
These actions affect the website’s back-end or front-end. Use a button.

2)

Actions where users won’t affect the website at all.
For example:
  • Browsing through content
  • Clicking and viewing pages
These actions do not affect the website’s back-end or front-end. Use a link.
